=====================================
2.1.2 Using a custom bitmap font
=====================================
This function allows you to use a bitmap font with custom character size or layout.
It should be called before initializing the root console with initRoot.

C++ : static void TCODConsole::setCustomFont(const char *fontFile,int charWidth=8, int charHeight=8, int nbCharHoriz=16, int nbCharVertic=16, bool inRow=false,TCODColor keyColor=TCODColor::black);
C   : void TCOD_console_set_custom_font(const char *fontFile,int char_width, int char_height, int nb_char_horiz, int nb_char_vertic, bool chars_by_row, TCOD_color_t key_color);

ParameterDescription
fontFileName of a .bmp file containing the font.
charWidth,charHeightSize of a character in the bitmap file.
nbCharHoriz,nbCharWidthSize of the characters grid in the bitmap file. The bitmap width must be nbCharHoriz * charWidth. The bitmap height must be nbCharVertic * charHeight.
inRowIf true, the first ASCII characters are in the first row in the bitmap file. Else, they are in the first column.
keyColorColor in the bitmap that represent a transparent pixel.
Example :
7x7 characters stored in a 112x112 bitmap file (thus containing an array of 16x16 characters)

C++ : TCODColor keyCol(255,0,255); // key color = pink
      TCODConsole::setCustomFont("Herrbdog_7x7_tileset.bmp",7,7,16,16,true,keyCol);
C   : TCOD_color_t key_col={255,0,255};
      TCOD_console_set_custom_font("Herrbdog_7x7_tileset.bmp",7,7,16,16,true,key_col);
